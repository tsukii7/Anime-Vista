# 👥 项目分工说明文档

本项目共有四位成员（曾宪清、林雨航、杨若桐、刘政昊），采用 Model-View-Presenter 架构，结合 Firebase 提供后端支持，并依据前端原型图完成页面开发与逻辑实现。

本文档用于说明各成员具体职责、分工原则及对应任务列表，确保项目进度清晰、责任明确。

## 🎯 分工原则

1. 曾宪清 负责所有页面的 UI 原型绘制，因此需要承担页面样式一致性的检查与优化任务。
2. 林雨航 除页面任务外，还负责把控前后端整体架构，即 Firebase 后端支持与全部项目文档的撰写，以及项目进度。
3. 杨若桐 与 刘政昊 主要负责页面逻辑和组件开发，无需深入理解整体架构，重点在于代码实现。
4. 页面任务与额外职责综合评估工作量，力求每位成员工作量平均，约为 7 ~ 8 分。

## 📦 页面工作量评估

| 页面               | 说明                                  | 工作量（分） |
|------------------|-------------------------------------|--------|
| Home             | 推荐、热门展示，结构清晰                        | 3      |
| Current          | 包含 List 和 Timeline 两种视图，逻辑较复杂       | 4      |
| Search           | 多筛选条件、表单交互 React Hook Form，逻辑中等偏上   | 3.5    |
| Rank             | 条件 + 三种排序逻辑，结构简单                    | 2.5    |
| Me               | Favorites 与 Activities 双视图，涉及用户数据处理 | 3.5    |
| Details          | 组件数量最多，数据结构复杂                       | 5      |
| Login / Register | 表单处理 + Firebase Auth                | 1.5    |
| About            | 展示性页面，仅包含开发者信息卡片                    | 1.5    |

总页面工作量：24.5 分

总非页面工作量：7.5 分
